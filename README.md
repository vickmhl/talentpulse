# TalentPulse V6

TalentPulse 是一个面向 HR 学生与初级 HR 从业者的中文组织诊断作品。

它不是配置型后台，也不是普通分析看板，而是一套把员工数据转成可汇报组织判断、风险外推与动作建议的 PC 网页展示台。

核心主张：

`把员工数据转成可汇报的组织诊断。`

## V6 升级重点

V6 聚焦两件事：

1. 视觉与排版重构
2. 图表化表达重构

当前版本重点把产品从“很多卡片的分析站”收成“高质感中文组织诊断展示台”。

## 六个主页面

- 首页
- 总览
- 人才盘点
- 继任分析
- 数据健康
- 汇报报告

其中：

- 首页 / 总览 / 汇报报告：主讲页
- 人才盘点 / 继任分析 / 数据健康：证据页

## 双演示样本

### 官方案例

- 用于首页、总览和汇报报告
- 数据口径统一，不故意注入脏值
- 组织故事稳定，可直接演示

### AI 清洗演示版

- 用于上传演示和数据健康页
- 保留字段别名、部门别名、岗位简称、缺失与异常
- 组织故事与官方案例一致

## 官方案例公司

公司名：`澄曜科技`

核心组织故事：

- 研发中心：高潜储备不低，但现在可接任管理梯队偏薄
- 销售增长：当前业绩强，但头部依赖重、保留风险高
- 交付运营：运行稳定，但成长动能不足、后备薄
- 产品与设计：关键岗位集中，单点依赖明显
- 客户成功：团队规模不小，但高潜识别不足
- 人力资源 / 财务法务 / IT与数据平台：人数少但继任暴露高

## 当前分析框架

当前引擎分两步完成诊断：

1. 识别结构信号
2. 组合成组织问题

结构信号包括：

- 高潜集中但现在可接任偏薄
- 高绩效集中且高流失风险高
- 关键岗位无后备聚集
- 覆盖存在但深度不足
- 单点依赖明显
- 稳定但低高潜、低 1-2 年可接任储备
- 支持职能低可见度风险

## V6 页面重点

### 首页

- 只保留主张、入口和官方案例预览
- 用更强的 Hero 与缩略图表达代替说明型卡片
- 正式模式默认隐藏作者练习痕迹

### 总览

- 按 `公司现状 -> 部门现状 -> 组织问题 -> 风险外推 -> 建议动作` 讲述
- 部门对比可交互
- 问题、风险与动作联动
- 用图表和图形化证据替代大段说明

### 人才盘点

- 先看人才密度、伪强项和梯队断层
- 九宫格退到证据位
- 代表样本后置到抽屉

### 继任分析

- 先看岗位暴露结构
- 用无后备 / 弱覆盖 / 单点依赖三类暴露组织页面
- 岗位与候选人细节后置到抽屉

### 数据健康

- 首屏只讲可信度、识别、修复与谨慎项
- 表格下沉为附录
- 更像可信度说明页，而不是问题中心

### 汇报报告

- 更接近 executive one-pager
- 三个核心问题并列呈现
- 风险与动作按映射关系展示
